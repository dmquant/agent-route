核心设计原则

Anthropic 管理型智能体原则落地

架构遵循 Anthropic《构建有效智能体》研究的核心理念：

大脑是"牲畜"而非"宠物"

编排器崩溃后，wake(sessionId) 重放事件日志即可恢复。内存中无不可替代的状态。

Hand 调用工具，Brain 调用 Hand

编排器不知道底层是 CLI 进程还是 HTTP API — 只知道 Hand 接口。

上下文存在于 LLM 之外

所有上下文持久化存储在事件日志中。上下文引擎决定每轮注入哪些内容。

Harness 编码会过时的假设

每个智能体的配置（上下文限制、压缩、重试）是显式的，可以热替换而不中断会话。
